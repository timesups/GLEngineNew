二、第三方依赖接入（两种方案）
- 方案A（推荐）：vcpkg（最省事）
  - 安装 vcpkg 后在 VS 集成（环境变量 VCPKG_ROOT，或用“集成 MSBuild”）
  - 安装包：
    - vcpkg install glfw3 glm spdlog
    - imgui 两种策略：
      - 使用 vcpkg：vcpkg install imgui[core,glfw-binding,opengl3-binding]
      - 或选择“方案B”自带源码（更可控）
  - 链接 OpenGL：opengl32.lib（Windows 系统库）
- 方案B：vendor 到 external/
  - glad：建静态库项目 glad，源为 external/glad/src/glad.c，包含目录 external/glad/include，链接到 Engine
  - imgui：建静态库项目 imgui，添加
    - imgui*.cpp, imgui*.h
    - backends/imgui_impl_glfw.cpp/.h
    - backends/imgui_impl_opengl3.cpp/.h
    - Public Include：external/imgui，external/imgui/backends
    - 由 Engine 或 DemoApp 链接 imgui
  - stb：仅头文件 external/stb/stb_image.h；在 Engine/ThirdParty/stb_image_impl.cpp 定义实现
  - glfw/glm：可 vendor 或改用预编译库/自建项目（或仍用 vcpkg）

三、各项目（VS 工程）配置要点
- Engine（Static Library）
  - C/C++:
    - C++20、/W4、/permissive-、NOMINMAX、_CRT_SECURE_NO_WARNINGS
    - 预编译头可用 Pch.h
    - 包含目录：external/glad/include、external/imgui（仅当封装 ImGuiLayer 直接引用其头）、external/stb、external/glm（或 vcpkg 自动）
  - Linker: 无（静态库）
  - 依赖：glad、imgui、glfw、glm、spdlog、opengl32（opengl32 由应用层链接亦可）
  - 运行时：Debug 模式开启 KHR_debug（glDebugMessageCallback）
- imgui（Static Library，若选方案B）
  - 源文件：imgui*.cpp + backends/*.cpp（只保留 glfw+opengl3）
  - 包含目录：external/imgui; external/imgui/backends
  - 预定义：IMGUI_IMPL_OPENGL_LOADER_GLAD（使用 GLAD）
- glad（Static Library）
  - 源：external/glad/src/glad.c
  - 包含目录：external/glad/include
- DemoApp（Console Application）
  - 链接：Engine、imgui、glad、glfw3、opengl32、spdlog（可选）
  - Debugging → Working Directory：$(TargetDir)
  - Post-Build Event：复制 assets 到 $(TargetDir)assets
    - xcopy "$(SolutionDir)assets" "$(TargetDir)assets" /E /Y /I
  - 包含目录：必要的 external/*（若未通过 Engine 传递）

四、ImGui 集成位置与调用约定
- ImGuiLayer（Engine/ImGui/ImGuiLayer.h/.cpp）
  - 初始化：
    - ImGui::CreateContext()
    - ImGui_ImplGlfw_InitForOpenGL(window, true)
    - ImGui_ImplOpenGL3_Init("#version 450")
  - 每帧：
    - ImGui_ImplOpenGL3_NewFrame()
    - ImGui_ImplGlfw_NewFrame()
    - ImGui::NewFrame()
    - …绘制控件…
    - ImGui::Render()
    - ImGui_ImplOpenGL3_RenderDrawData(ImGui::GetDrawData())
  - 关闭：
    - ImGui_ImplOpenGL3_Shutdown()
    - ImGui_ImplGlfw_Shutdown()
    - ImGui::DestroyContext()

五、工作流与里程碑（VS 视角）
- M0：Engine + DemoApp 工程可编译，创建 GLFW 窗口，加载 GLAD，清屏。
- M1：集成 ImGui（显示 Demo Window），在 DemoApp 的渲染循环调用 ImGuiLayer。
- M2：封装 VAO/VBO/EBO + Shader，渲染立方体；引入 Camera 和 Input。
- M3：stb_image 纹理加载，sRGB 纹理与帧缓冲设置；简单光照。
- M4：FBO + 后处理（Tonemap）。
- M5：资源管理与热重载（可选），阴影或 PBR（二选一）。

六、glad.c 与 stb_image.h 放置（与上文一致）
- external/glad/src/glad.c（独立静态库 glad）
- external/stb/stb_image.h（仅头文件）
- src/Engine/ThirdParty/stb_image_impl.cpp 中定义一次：

  - #include <stb_image.h>
